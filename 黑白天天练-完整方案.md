### 1.3 SGF 坐标规则

```
字母映射：a=0, b=1, c=2, d=3 ... s=18
SGF坐标：第一个字母=列(x)，第二个字母=行(y)
AB[ce] → 黑子在 (2, 4)
AW[df] → 白子在 (3, 5)
SZ[19] → 19路棋盘
SZ[9]  → 9路棋盘
```

**必须先跑验证测试：**
```
解析 (;SZ[9]AB[ee]AW[ff])
期望结果：黑子在(4,4)即9路棋盘正中央，白子在(5,5)
如果坐标不对，修正映射逻辑后再继续
```

### 1.4 正确分支判断

SGF 题目中正确答案的识别规则（按优先级）：
1. 含 `TE[1]` 或 `TE[2]` 的变化分支 = 正解
2. 含 `C[Correct]` 或 `C[RIGHT]` 的节点 = 正解
3. 以上都没有时，取主分支（第一个变化）的第一手作为正解

### 1.5 输出格式

每道题输出为 JSON：

```json
{
  "id": "sanderland_cho_elementary_001",
  "source": "sanderland/tsumego",
  "source_file": "problems/cho/elementary/001.sgf",

  "board_size": 19,
  "initial_stones": {

```

所有题目保存为 `problems_all.json`。

### 2.2 ELO 计算公式

每做一道题，相当于和题目"对弈"一局：

```javascript
// R_user: 用户当前等级分
// R_problem: 题目难度分
// K = 10

// 期望胜率
const E = 1 / (1 + Math.pow(10, (R_problem - R_user) / 400));

// S = 1（做对），S = 0（做错）
const R_new = R_user + K * (S - E);

// 举例：用户360分做360分的题
// 做对：360 + 10 * (1 - 0.5) = +5
// 做错：360 + 10 * (0 - 0.5) = -5

// 用户360分做450分的题（高3级）
// 做对：+6.4
// 做错：-3.6
```

### 2.3 保护机制

- 等级分最低为 0，不会变成负数
- 每天等级分变动上限 ±50（防止一天刷太多导致虚高或崩盘）
- 刚升级后有"保护期"：升级后前 5 题做错不扣分（避免刚升就掉的挫败感）
- 连续做错 5 题，系统自动降低出题难度
- 做错同一道题第二次，不再扣分

## 三、出题策略

### 3.1 难度匹配

系统根据用户当前等级分选题：

| 比例 | 题目难度范围 | 目的 |
|------|------------|------|
| 70% | 用户等级分 ±30 | 同级题，稳定涨分 |
| 20% | 用户等级分 +30~+90 | 挑战题，做对涨更多 |
| 10% | 用户等级分 -30~-90 | 简单题，保持信心 |

目标正确率控制在 70%-80%（参考 Duolingo 的 Birdbrain 系统）。

### 3.2 去重

- 记录用户已做过的题目 ID
- 优先出没做过的题
- 全部做完一轮后，优先出错题本里的题（间隔重复）
- 同一天内不出重复题

### 3.3 题型标签

根据 SGF 内容自动标注（如果原始数据有的话）：
- 死活题（黑先活/黑先杀白）
- 手筋题
- 吃子题
- 官子题

如果无法自动判断，统一标注为"死活题"。

## 四、每日目标与奖励

### 4.1 每日目标

用户首次进入时选择每日目标（可随时修改）：

| 目标档位 | 每日题数 | 吉祥物文案 |
|---------|---------|-----------|
| 轻松 | 3 题 | "3题就好，轻轻松松~" |
| 标准 | 5 题 | "5题刚刚好！" |
| 认真 | 10 题 | "今天要加油哦！" |
| 疯狂 | 20 题 | "你是真的猛！" |

默认选中"标准（5题）"。

### 4.2 做对奖励（三层）

**第一层：即时反馈（每道题做对后）**
- 吉祥物小黑蹦出，随机显示一条夸奖文案：
  - "一步到位！"
  - "秒杀！"
  - "厉害了！"
  - "这都能看到？"
  - "妙手！"
  - "读棋高手！"
  - "准确！"
  - "小黑佩服！"
- 等级分 +5（同级题，具体数值由 ELO 公式算）
- 显示 "+X" 得分动画
- 连对加成：连续做对 N 题额外加分
  - 连对 2 题：额外 +1
  - 连对 3 题：额外 +2
  - 连对 5 题：额外 +3，显示"🔥 火力全开！"
  - 连对 10 题：额外 +5，显示"🔥🔥 无人能挡！"

**第二层：完成每日目标**
- 小黑变星星眼，播放撒花/撒星星粒子动画
- 打卡日历点亮今天（绿色对勾✅）
- 获得 1 个"棋子币"（虚拟货币，后期用于兑换皮肤）
- 今日目标进度条上显示 "今日目标已完成！"

**第三层：长期激励**
- 连续打卡勋章：
  - 连续 3 天 → "入门棋手" 称号
  - 连续 7 天 → 铜勋章 🥉
  - 连续 30 天 → 银勋章 🥈
  - 连续 100 天 → 金勋章 🥇
  - 连续 365 天 → 钻石勋章 💎
- 升级庆祝：升级时全屏庆祝动画，小黑小白一起跳舞
- 里程碑成就：
  - 做满 100 题 → "百题斩"
  - 做满 500 题 → "五百强"
  - 做满 1000 题 → "千题王"
  - 首次升段（到1D）→ "初段达成"

### 4.3 做错处理

**核心原则：轻处罚、重引导，绝不让用户产生挫败感。**

**即时反馈：**
- 不显示刺眼的"❌ 错误"
- 小黑说温柔的话，随机选一条：
  - "没关系，看看正解~"
  - "差一点点！"
  - "这题确实有点难"
  - "想想再试试？"
  - "别灰心，下一题！"
- 棋盘上自动演示正确走法动画（把正解路线走一遍，1-3手，每手停顿0.5秒）
- 等级分 -5（同级题，具体由 ELO 公式算）

**错题本机制：**
- 做错的题自动进入"错题本"
- 隔 1 天后重新出现（间隔重复）
- 如果第二次做对，从错题本移除
- 如果第二次还是做错，间隔延长到 3 天后再出
- 第三次还错，间隔延长到 7 天
- 错题本中的题做错不扣等级分（只有首次做错扣分）

**难度自动调整：**
- 连续做错 3 题：下一题自动降低难度（出低 1 级的题）
- 连续做错 5 题：降低 2 级，并且小黑说"换几道简单的练练手~"
- 用户主动点"太难了"按钮：立即降低出题难度

## 五、天梯首页

### 5.1 节点设计

天梯路径上的节点 = 级别，从底部 25K 到顶部 5D，共 30 个节点。
S 形蜿蜒排列，每屏显示 5-7 个节点。

节点状态：
- ✅ 已通过（绿色，显示对勾）
- 🟠 当前等级（橙色，高亮，显示等级数字，有呼吸动画）
- 🔒 未解锁（灰色，显示锁图标）

当前等级的节点旁边站着吉祥物小黑，带对话气泡说：
- "还差 X 分升级！"
- "今天做了 Y 题，继续加油！"
- "连续打卡 Z 天了！"

### 5.2 背景分层

从底部到顶部：
- 25K-20K：草地 + 小花 + 白天蓝天
- 20K-15K：丘陵 + 白云
- 15K-10K：云层之上 + 黄昏
- 10K-5K：星空 + 月亮
- 5K-1D：深邃星空 + 银河
- 1D-5D：金色星云 + 顶部光芒

### 5.3 顶部信息栏

显示：
- 🔥 连续打卡天数
- 当前等级（如"15级"）
- 棋力分数（如"912"）
- 今日目标进度（3个圆点，做完一题亮一个）
- "开始"按钮
